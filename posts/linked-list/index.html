前言#

前陣子在找工作也花了蠻多時間寫leetcode，在leetcode有許多關於linked-list的題目，討論區可以看到不同思維的解法，其中有一種思維很有趣，近期在jserv老師的課你所不知道的C語言: linked list 和非連續記憶體操作看到非常詳細的解釋，於是想藉由刷過的題目紀錄一下這種思維，由於老師的筆記太過詳盡，所以我打算直接以題目的角度切入作為補充。

linked-list#

Linked list作為一個基本的資料結構，對於學習程式的人並不陌生，但其實更深入探討會發現linked list可以玩味之處眾多。在leetcode的題目中，多數此類題目的資料結構為singly linked list，對應的程式碼如下：

C:

typedef struct ListNode {
    int val;
    struct ListNode* next;
} node_t;

Go:

type ListNode struct  {
    Val int
    Next *ListNode
}

以三個節點：A, B, C 單向連結的list為例，每個節點都會指向一個節點的記憶體位址：

leetcode 21. Merge Two Sorted Lists#

題目描述：給定兩個已排序的linked list，回傳合併過後的linked list。

Merge two sorted linked lists and return it as a sorted list. The list should be made by splicing together the nodes of the first two lists.

這題在討論區較常見的iterative作法為新增一個dummy node，依序將值較小的節點串上，最後回傳dummy的下一個。範例程式碼：

C:

struct ListNode* mergeTwoLists(struct ListNode* l1, struct ListNode* l2){
    typedef struct ListNode node_t;
    node_t *head = (node_t*)malloc(sizeof(node_t));
    node_t *ptr = head;
    while (l1 && l2) {
        if (l1->val < l2->val) {
            ptr->next = l1;
            l1 = l1->next;
        } else {
            ptr->next = l2;
            l2 = l2->next;
        }
        ptr = ptr->next;
    }
    ptr->next = l1 ? l1 : l2;
    return head->next;
}

Go:

func mergeTwoLists(l1 *ListNode, l2 *ListNode) *ListNode {
	head := &ListNode{0, nil}
	ptr := head

	for l1 != nil && l2 != nil {
		if l1.Val <= l2.Val {
			ptr.Next = l1
			l1 = l1.Next
		} else {
			ptr.Next = l2
			l2 = l2.Next
		}
        ptr = ptr.Next
	}
	if l1 != nil {
		ptr.Next = l1
	} else {
		ptr.Next = l2
	}
	return head.Next
}

可以發現此種解法最後都是回傳第二個節點，這樣的作法並無不妥，時間及空間的表現上都是最優解了，但如果不想新增一個無用的節點有沒有其他作法？以前面提到的ABC節點為例，如果要將B節點刪除，此題的思維會更改A節點的Next值：

C: A->next = A->next->next Go: A.Next = A.Next.Next

換個思維，與其將A的Next值，不如改變存在Next位址的值，也就是把C放進0x02a7的位址：

這樣indirect pointer的思維可以研讀jserv老師的筆記，如果將這種思維應用在這題程式碼如下：

C:

struct ListNode* mergeTwoLists(struct ListNode* l1, struct ListNode* l2){
    typedef struct ListNode node_t;
    node_t *head;
    node_t **ptr = &head;
    while (l1 && l2) {
        if (l1->val < l2->val) {
            *ptr = l1;
            l1 = l1->next;
        } else {
            *ptr = l2;
            l2 = l2->next;
        }  
        ptr = &((*ptr)->next); // 更新至下次要存入的位址
    }
    *ptr = l1 ? l1 : l2;
    return head;
}

Go:

func mergeTwoLists(l1 *ListNode, l2 *ListNode) *ListNode {
    head := new(ListNode)
    ptr := &head
    for l1 != nil && l2 != nil {
        if l1.Val < l2.Val {
            *ptr = l1
            l1 = l1.Next
            ptr = &((*ptr).Next)
        } else {
            *ptr = l2
            l2 = l2.Next
            ptr = &((*ptr).Next)
        }
    }
    if l1 == nil {
        *ptr = l2
    } else {
        *ptr = l1
    }
    return head
}

結論#

這類將值放入某個位址的思維有別於以往我們熟悉的，直接更新值指向的位址，也許有些人認為多此一舉，但我覺得學習語言有時是在學習一種文化，一如我們學習自然語言一樣，不僅僅是語法，而是當地人的習俗、特色以及思考方式，因此這種思維也是值得我們去探究的。
